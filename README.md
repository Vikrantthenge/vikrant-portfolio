### ✈️ FlightPulse — Flight Delay Early-Warning System

**The operational problem:** Turnaround managers need to know *which flights are most likely to slip* before the delay happens — not after. Standard reporting tells you what went wrong; this tells you what is about to go wrong.

**What it does:** Classifies incoming flights by delay risk and surfaces the specific operational drivers behind each prediction — so the manager knows whether the issue is a crew rotation, a gate conflict or an engineering hold, and can act accordingly.

**Tools:** Python · Machine Learning · SHAP model explainability · Decision-support output design

### 👥 CrewOps — Workforce & Roster Scenario Planner

**The operational problem:** Roster decisions made on gut feel or historical patterns consistently produce gaps — overtime spikes, coverage failures and avoidable disruption. Managers need to stress-test a proposed roster against demand *before* committing to it.

**What it does:** Lets an operations manager model multiple staffing scenarios against shift demand, identify coverage gaps and compare resource-allocation options — before a single person is rostered. Built on the same workforce-planning principles used in airline crew scheduling.

**Tools:** Python · Scenario modelling · Workforce analytics · Streamlit

### 📊 Workforce & Capacity Forecasting Tool

**The operational problem:** Reactive staffing — responding to demand rather than anticipating it — is one of the most persistent cost and service problems in airline and airport operations. Planning against a static schedule is not enough when demand patterns shift.

**What it does:** Generates forward-looking demand forecasts and capacity assessments using operational-planning principles drawn from airline scheduling, enabling proactive resource allocation rather than reactive firefighting.

**Tools:** Python · Demand forecasting · Capacity planning · Power BI · Streamlit

### ⚠️ Operational Risk & KPI Performance Monitor

**The operational problem:** KPI dashboards typically show what has already happened. Operations managers need a system that surfaces *emerging* risks — trends that are deteriorating before they breach a threshold — so they can intervene before the incident, not after.

**What it does:** Monitors operational-performance trends in real time, calculates variance from target, identifies early-warning signals and presents a management-ready view of where risk is building across the operation.

**Tools:** Python · SQL · Power BI · KPI dashboard design · Trend and variance analysis

### 🤖 AI Operations Intelligence Assistant

**The operational problem:** Aviation safety and operations data is abundant but unstructured. Compiling it into management-ready summaries is slow, manual and often happens too late to influence decisions. Sending operational data to an external cloud service creates security and confidentiality risk.

**What it does:** A locally-hosted Generative AI assistant — deployed using Ollama, Docker and Open WebUI — that analyses aviation safety and operational data and produces structured, executive-style management summaries without sending data to an external provider. Demonstrates how AI can be deployed inside an airline's own environment for real-time operational intelligence.

**Tools:** Generative AI · Ollama · Docker · Open WebUI · Prompt engineering · Local LLM deployment
